# 🛡️ هندسة وكيل الويب (Thor Web Agent - Envoy Sidecar)
**التقنيات المعتمدة بناءً على البنية الهيكلية الجديدة**

بناءً على اعتماد بنية (Sidecar) مع وكيل (Envoy Proxy)، تنجلي الرؤية لتكون أكثر استقراراً وقابلية للتوسع في بيئات السحابة و Kubernetes. التخلي عن Uprobes لصالح Envoy Proxy مع إنهاء التشفير (TLS Termination) يعطينا تحكماً موثوقاً وأداءً قياسياً.

---

## 1. طبقة النواة السريعة (XDP Kernel Layer)
*الجدار الفولاذي الأول قبل وصول الحزم المعقدة إلى الويب.*
*   **التقنية:** `eBPF / XDP` مبرمج بـ C/Rust.
*   **الوظيفة:** فلترة مسبقة بسرعة الأجهزة (100 نانوثانية).
    *   **IP Reputation:** إسقاط أي اتصال من IPs خبيثة (تأتي القائمة السوداء من SOC).
    *   **Rate Limiting:** تطبيق خوارزمية (Token Bucket) محلياً لتقييد الـ IPs التي تستهلك الباندويث.

## 2. وكيل الشبكة والويب (Envoy Proxy Sidecar)
*المستقبل الأساسي للطلبات وإنهاء التشفير.*
*   **التقنية:** `Envoy Proxy` (مفتوح المصدر من CNCF، مكتوب بـ C++ ويدعم الأداء الخارق).
*   **الوظيفة:**
    *   إنهاء التشفير (TLS Termination) لمعالجة الطلبات محلياً.
    *   دعم بروتوكولات (HTTP/1, HTTP/2, HTTP/3).
    *   GeoIP Filtering لصد الطلبات بناءً على الدولة.

## 3. محركات الحماية الأساسية والتحليل (Thor Engine + Coraza)
*دماغ الوكيل المحلي الذي يحلل الطلبات المفكوكة التشفير خلال 1-3ms.*
*   **التقنية 1 (القواعد التقليدية):** `Coraza WAF` مدمج مع Envoy (عبر WASM أو ext_authz). هذا يحمي من هجمات OWASP عبر قواعد (ModSecurity) مجربة وسريعة.
*   **التقنية 2 (التحليل الدلالي - Thor Engine):** كود `Rust` يعمل كخدمة مرافقة لـ Envoy ويشغل نماذج ذكاء اصطناعي خفيفة (ONNX بوزن 10-50MB) لفحص الـ Payloads التي تجاوزت Coraza لاكتشاف الـ Zero-day.
*   **التقنية 3 (التحقق من الـ API):** `API Schema Validation` لضمان أن هيكل طلبات الـ HTTP يطابق الـ OpenAPI Swagger الخاص بالتطبيق.

## 4. التحليل السلوكي المحلي (Local Behavioral Analysis)
*   **التقنية:** `Redis` محلي مدمج في الـ Sidecar.
*   **الوظيفة:** تتبع سلوك المستخدم (مثلاً، عدد محاولات تسجيل الدخول الخاطئة، وتيرة الطلبات المعقدة) دون الحاجة للاتصال بالـ SOC، مما يوفر استجابة لحظية في الوكيل.

## 5. الاتصال بمركز العمليات (Out-of-band SOC Integration)
*   **التقنية:** `gRPC` over `mTLS` مع `gRPC-Web`.
*   **الوظيفة:** إرسال **عينة/ملخصات ذكية للطلبات المشبوهة** فقط إلى الـ SOC المركزي ليقوم بالتحليل الأعمق (الذي يستغرق 20-50ms باستخدام نماذج SLM 7B)، مع استقبال التحديثات اللحظية للقوائم السوداء لتغذية XDP و Envoy.

---

### ملخص سير العملية الجديد للطلبات (Data Flow):
1. **[XDP / eBPF]** يستقبل الحزمة. إذا كان الـ IP في القائمة السوداء، تُسقط فوراً (0.0001ms).
2. **[Envoy Proxy]** يستقبل الطلب السليم من النواة ويقوم بإنهاء التشفير (TLS Termination).
3. **[Coraza WAF / WASM]** يفحص الطلب بسرعة ضد قواعد ModSecurity معروفة (SQLi, XSS).
4. **[Thor Engine / Ext_Authz]** يحلل محتوى الطلب المتجاوز بواسطة الـ (Small AI Model 10-50MB) ويطابق الـ API Schema ويقارنه بتخزين (Redis) للسلوك (Takes 1-3ms).
5. **[Decision]** إما أن يُمرر الطلب لتطبيق الويب، أو يُسقط.
6. **[Async Logger]** يُرسل ملخص الحدث الشاذ إلى الـ SOC لمعالجته بنموذج كبير (7B Parameter) خارج مسار الطلب الأساسي دون تعطيل المستخدم.
